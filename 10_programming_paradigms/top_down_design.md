# Top-Down Design

Top-down design means:
- starting with the main problem
- then breaking it into smaller sub-problems

---

# Example

## Main Goal
Create a quiz game.

↓

## Smaller Problems
- Display questions
- Accept input
- Check answers
- Keep score

↓

## Even Smaller Problems
- Print messages
- Compare answers
- Add points

---

# Advantages

- Easier planning
- Easier debugging
- Easier teamwork
- Easier maintenance
